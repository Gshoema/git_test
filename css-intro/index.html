Syntax

Universal Selector

* { color: purple; }

Type Selectors (AKA Element Selectors)

<!-- index.html -->
Hello, World!
Hello again!

Hi...

Okay, bye.

/* styles.css */ div { color: white; }

Type (Element) Selectors apply to all html elements of a known type. In this case all the 'div' type elements will be effected, and not the 'p' type elements.

Class Selectors

<!-- index.html -->
Please agree to our terms of service.

/* styles.css */ .alert-text{ color: red; }

The Class Selector will select all elements with a given class. A class is an attribute you place on an html element (alert-text). The syntax for class Selectors and all selectors cannot be named with a leading number. For example 4lert-text would fail in the scenario above.

another thing with class attributes is that you can add multiple classes to a single element. class="alert-text severe-alert" is totally legal syntax. This also shows that whitespace is used in ways to separate classes and you should always give names a hyphen instead of spaces.

ID Selectors

<!-- index.html -->
My Awesome 90's Page
/* styles.css */ #title{ background-color: red; }

Instead of a period, ID's use a hashtag. It is a common pitfall to use the ID attribute when classes will suffice. ID's should be used sparingly and for specific use cases such as links redirecting to a section on the current page, or taking advantage of specificity. Leading Digits are also illegal with ID syntax.

Grouping Selectors

.read { color: white; background-color: black; /* several unique declarations */ } .unread { color: white; background-color: black; /* several unique declarations */ }

both .read and .unread share color and background-color selectors. These can be comma separated as follows:

.read, .unread { color: white; background-color: black; } .read { /* several unique declarations */ } .unread { /* several unique declarations */ }

Both examples will work and have the same result, but the second example reduces repetition of declarations and makes it easier to edit both classes at once.

Chaining Selectors

<div class="subsection header">Latest Posts</div> <p class="subsection preview">This is where a preview for a post may go.</p> .subsection.header{ color: red; }

What .subsection.header does is it selects any element that has both subsection AND header classes. This syntax can be used to chain any combo of selectors, except Chaining more than one Type Selector.

You can chain classes and an ID, for example the preview selector where an ID selector:

<p class="subsection" id="preview">This is where a preview may go</p> .subsection#preview{ color: blue; }

Descendant Combinator

Cominators allow us to combine selectors differently than grouping or chaining, and they show a relationship between selectors. There are 4 types of combinators, but for now we'll go into the Descendant Combinator.

Descendant Combinator is represented as a single space between selectors. classes are nested within eachother with the topmost class being the 'ancestor' and classes neseted within the ancestor are called 'children'. For example:

<div class="ancestor">

     <div class="child">

While the css would look like the following:

.ancestor .child{ Some Declarations }
